Web Interface of Priority DMS

«Priority» is a document management system used by Sberbank, Russian Post, and many other companies in Russia. As an addition to the good old desktop version, I designed a light web interface for it.

Employees create documents and send them to their managers for approval. Managers approve documents, sign them with electronic signatures, and check what how their subordinates are doing. The UI is designed so both the management and regular contributors easily find the functions they need.

The new web UI replaced the Windows app for many users, so it was necessary to preserve the basic logic and familiar elements, but make the UI friendly and modern. I’ve introduced several solutions to simplify dull work and prevent human errors: auto-saving, soft delete, hints in forms.

Official document management is strictly regulated, so it’s quite easy to predict most users’ actions. When possible, the system works instead of a human: attaches related documents, marks requests with reports as completed, and calculates approval deadlines.

The system keeps evolving, and new features are added from time to time. The UI is built from blocks, so it's possible to expand it without rebuilding. To help the developers seamlessly add new blocks, I created a simple UI kit. It describes atomic elements and principles of block building.
